# Introduction

**xtan.ai** explores a new approach to **precision gesture interaction for professional 3D workflows**.

The project focuses on enabling **reliable, low-latency spatial interaction** for applications such as CAD, digital content creation (DCC), robotics, and VR authoring. Instead of relying purely on AI-based depth inference, the architecture builds on **metric stereo geometry**, providing stable 3D keypoints that can be interpreted consistently over time.

The system is composed of several layers:

* **EdgeTrack** – stereo capture hardware and edge-side processing
* **CoreFusion** – multi-view fusion and reconstruction of stable 3D keypoints
* **MotionCoder** – interpretation layer that converts motion into structured interaction commands

This modular architecture separates **capture, reconstruction, interpretation, and application integration**, allowing the same interaction system to be reused across many professional tools.

The primary users are **3D creators, developers, robotics researchers, and technical artists** who require reliable spatial input beyond traditional mouse/keyboard interfaces.

---

# What is MotionCoder

MotionCoder receives **time-consistent 3D keypoints** from CoreFusion (generated by EdgeTrack) and translates them into **structured gesture commands**.

It converts precise 3D motion into **repeatable interaction events** suitable for professional workflows such as:

* 3D authoring
* VR creation tools
* robotics interfaces
* spatial design environments

Unlike consumer hand-tracking systems that mainly prioritize **visual appearance**, MotionCoder focuses on:

* spatial accuracy
* timing consistency
* reproducibility

The goal is not simply to *track hands*, but to enable **controlled interaction with digital tools**.

---

# AI Assist (Lightweight, Purpose-Built)

MotionCoder includes a lightweight **AI assist layer** that processes the reconstructed 3D keypoints and performs:

* gesture classification
* confidence estimation

The models are **graph-based neural networks** trained using supervised and self-supervised learning. They are specifically designed for motion data and **do not rely on large language models**.

Because the input already consists of stable 3D keypoints, the models can run efficiently on **modest GPU hardware** (for example CUDA-capable GPUs) and do not require datacenter-class accelerators.

# How This Differs from Typical AI-Only Gesture Recognition

Many AI gesture systems rely heavily on **per-frame pose classification**.

This approach can work well for broad signals (for example a simple wave), but it often fails when meaning depends on **motion over time, timing, or context**.

For example, a flat hand (🖐️) can represent several very different actions depending on how it moves:

* **Hello** – repeated waving motion
* **Stop** – static pose held in place
* **Push / slide** – directional motion with speed and interaction context

Pose-only classifiers often collapse these situations into the same label and can be sensitive to real-world variations such as:

* finger articulation differences
* viewpoint changes
* partial occlusion
* motion blur
* background clutter

MotionCoder addresses this by combining **temporal motion analysis**, **gesture grammars**, and **context-aware state machines**.

Instead of classifying single frames, the system evaluates:

* how motion evolves over time
* how stable a pose is
* what happened immediately before
* what tool or mode is currently active

This makes it possible to interpret gestures with much higher reliability and semantic precision.

# Mechanical Analogy

A useful analogy is a **mechanical control system**.

MotionCoder acts like the **cockpit**, translating human input into structured commands. But precise control requires a stable and balanced engine underneath.

In this system:

* EdgeTrack provides the **sensors**
* CoreFusion provides the **engine**
* MotionCoder provides the **control interface**

Just as a multi-cylinder radial engine delivers smoother and more stable power than a single-cylinder system, **multi-view geometry combined with temporal interpretation produces far more stable interaction** than frame-by-frame AI classification alone.

# MotionCoder – Primary Use Case: 3D Authoring

MotionCoder is primarily designed for **3D authoring workflows**.

The focus is **deep integration with existing software** such as:

* Blender
* Unreal Engine
* CAD systems

Rather than acting as a standalone input gadget, MotionCoder aims to provide **structured spatial interaction inside professional pipelines**.

In VR or desktop-based 3D environments, users can manipulate:

* objects
* geometry
* tools
* scene layouts

using **precise gestures or tracked physical tools**.

The emphasis is on **repeatable, context-aware interaction**, not purely visual tracking.
